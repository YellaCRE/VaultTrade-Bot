# 📌 1. 프로젝트 개요

## 1.1 프로젝트명

**VaultTrade Bot**

– 원금 보호를 최우선 가치로 하는 자동 매매 시스템

## 1.2 프로젝트 목적

- 업비트 Open API를 활용한 자동 매매 시스템 구축
- **수익 극대화가 아닌 “원금 보호”가 최우선**
- 시스템 리스크 / 시장 리스크 / 운영 리스크를 통제 가능한 구조로 설계

---

# 📌 2. 핵심 철학 (Core Philosophy)

### 1️⃣ 절대 수익 보장이 아닌, 손실 통제 구조 설계

현물 시장에서 “절대 무손실”은 수학적으로 보장 불가.

따라서 본 시스템은:

> **손실 가능성을 구조적으로 제한하는 시스템**
> 

을 목표로 한다.

---

### 2️⃣ 전략보다 리스크가 우선

- 전략은 수익을 시도한다.
- 리스크 관리 모듈은 손실을 강제로 제한한다.
- 리스크 모듈은 전략보다 상위 권한을 가진다.

---

### 3️⃣ 원금 보호 모드 정의

원금은 KRW 기준 총 자산으로 정의.

이를 다음과 같이 분리:

| 구분 | 설명 |
| --- | --- |
| Principal Vault | 절대 매매에 사용하지 않는 원금 |
| Trading Float | 매매에 사용 가능한 금액 |
| Equity | 전체 평가 자산 |

Trading Float만 매매에 사용하며,

Vault는 어떠한 상황에서도 사용 금지.

---

# 📌 3. 서비스 범위

## 3.1 In Scope

- 업비트 API 연동
- 단일 전략 자동매매
- 리스크 관리 시스템
- 실시간 모니터링 대시보드
- 로그 및 감사 기록 저장
- 알림 시스템

## 3.2 Out of Scope (초기 단계)

- 레버리지/선물
- 고빈도 매매
- 다거래소 차익거래
- 인공지능 학습 전략

---

# 📌 4. 시스템 목표

### 4.1 정량 목표

- 일 최대 손실 한도: X%
- 누적 최대 손실 한도: Y%
- Trading Float 비율: 전체 자산의 1~5%

### 4.2 안정성 목표

- API 장애 시 자동 거래 중단
- 주문 실패 시 자동 복구
- 모든 의사결정 로그 기록
- 비상 정지(E-Stop) 기능 제공

---

# 📌 5. 전략 설계 방향 (원금 보호 중심)

## 전략 원칙

1. 대형 코인만 거래 (BTC, ETH 등)
2. 소액 분할 매매
3. 보유 시간 제한
4. 변동성 급증 시 자동 중단
5. 수수료 포함 기대값 계산 후 진입

---

## 전략 유형 후보

### 1️⃣ 메이커 기반 스프레드 전략

- 지정가 매수 → 지정가 매도
- 작은 수익 반복
- 급변 시 주문 철수

### 2️⃣ 평균회귀 전략 (보수형)

- 단기 과매도 구간 진입
- 짧은 반등 구간 청산
- 단계 제한 필수

### 3️⃣ 수익 재투자 모드

- 최초 원금은 보존
- 수익 발생분만 재투자

---

# 📌 6. 리스크 관리 설계 (핵심)

## 6.1 손실 통제 정책

| 항목 | 설명 |
| --- | --- |
| Daily Loss Limit | 하루 손실 상한 |
| Total Drawdown Limit | 누적 손실 상한 |
| Max Position Size | 최대 보유 금액 제한 |
| Max Holding Time | 최대 보유 시간 제한 |

---

## 6.2 자동 중단 조건

- 일 손실 한도 초과
- 급격한 변동성 감지
- API 통신 오류 반복 발생
- 데이터 수신 지연
- DB 기록 실패

---

## 6.3 주문 안전 장치

- 미체결 주문 타임아웃 취소
- 중복 주문 방지
- 가격 괴리 감지
- 슬리피지 허용 범위 제한

---

# 📌 7. 운영 정책

## 7.1 기본 운용 모드

| 모드 | 설명 |
| --- | --- |
| Safe Mode | Trading Float 1% 이하 |
| Standard Mode | Trading Float 3~5% |
| Observe Mode | 실제 주문 없이 시뮬레이션 |

---

## 7.2 알림 체계

- 주문 체결
- 손실 한도 근접
- 자동 중단 발생
- 서버 장애
- 전략 비정상 종료
